# Pack-Renderer 세션 루틴

> 목적: Pack-Renderer 프로젝트의 세션 시작·종료 표준 절차
> 호출 시점: 새 세션 시작 시 즉시 / 종료 신호("종료·수고했어·오늘 여기까지") 수신 시
> 최종 업데이트: 2026-04-17 (v1)

순서를 임의로 바꾸거나 단계를 건너뛰지 말 것.

---

## 세션 시작 루틴 (0단계 포함 총 8단계)

**0단계 — graphify 지식 그래프 읽기 (최우선 필수)**
`graphify-out/GRAPH_REPORT.md`를 즉시 읽어라. 신 노드(god nodes)와 커뮤니티 구조를 파악한 후 이후 단계를 진행하라. 파일이 없으면 사용자에게 알리고 계속 진행하라.

**1단계 — 현재 위치 확인**
작업 디렉토리를 확인하라. 예상 경로(`/sessions/lucid-amazing-ride/mnt/Pack-Renderer`)가 아니면 사용자에게 알리고 진행 여부를 물어라.

**2단계 — state/claude-progress.txt 읽기**
지난 세션에서 무엇을 했는지, 어디까지 됐는지, 다음에 무엇을 해야 하는지 파악하라. 파일이 없으면 사용자에게 알려라.

**3단계 — state/feature_list.json 읽기**
`passes: false`인 항목을 확인하라. 이것이 남은 작업 목록이다. 파일이 없으면 사용자에게 알려라.

**4단계 — 원칙 파일 읽기 (작업 종류에 따라 필수)**
이번 세션에서 할 작업 종류를 파악하고, 해당 파일을 반드시 읽어라. 읽지 않고 코드 작성 금지.

| 작업 종류 | 읽어야 할 파일 |
|---|---|
| 설계 검증, 원칙 체크, validator 구현 | `skills/nickel_plate_principles.md` (LAYER 0·1·2) |
| 형상 생성, 알고리즘 구현, generator 구현 | `skills/nickel-plate-design.md` (LAYER 3) |
| SVG 렌더링, 색상·레이어·표시 구현 | `skills/nickel_render_rules.md` (LAYER 4) |
| 작업 종류 불명확 | 3개 파일 모두 읽기 |

**5단계 — git log 확인**
최근 커밋 20개를 훑어라 (`git log --oneline -20`). 마지막 커밋 이후 변경 사항을 파악하라.

**6단계 — init.sh 실행 (조건부)**
프로젝트 루트에 `init.sh`가 존재하고 개발 서버를 켜야 하는 상황이면 실행하라. 파일이 없거나 서버가 불필요하면 건너뛰어라.

**7단계 — 기본 기능 테스트 (조건부)**
실행 가능한 앱이 존재하면(HTML/서버 등) 정상 작동 여부를 확인한 후 작업을 시작하라. 앱이 없거나 테스트가 불필요하면 건너뛰어라.

모든 단계 완료 후, 사용자에게 현재 상태를 한 문장으로 요약하고 다음 작업을 제안하라.

## 세션 종료 루틴

사용자가 "종료", "다 됐어", "오늘은 여기까지", "수고했어" 등 종료 신호를 보내면 반드시 아래 순서대로 실행하라. 모두 완료하기 전까지 세션을 끝내지 말 것.

**1단계 — state/claude-progress.txt 업데이트**
오늘 한 일, 현재 상태, 다음에 할 일을 덮어써라.

**2단계 — state/feature_list.json 업데이트**
이번 세션에서 완료된 기능은 `passes: true`로 변경하라.

**3단계 — git 상태 점검**
1. `git status` — 미커밋 변경사항 확인. 있으면 사용자에게 알리고 커밋 여부를 물어라.
2. `git log origin/main..HEAD` (또는 `git status -sb`) — push되지 않은 로컬 커밋 확인. 있으면 사용자에게 명시하고 push 여부를 물어라. 사용자가 거부하면 그대로 종료하라.

**4단계 — 종료 요약 출력**

```
오늘 작업 요약
──────────────────────────────
한 일     : (핵심 작업 2~3줄)
미완료    : state/feature_list.json passes:false 개수
git 상태  : 커밋 완료 / push 완료 여부
다음 작업 : (1순위 항목)
──────────────────────────────
```

---

## 조기 종료 방지

`state/feature_list.json`의 `passes: false` 항목이 남아 있는 한 "다 됐다"고 선언하지 말 것.
반드시 미완료 항목을 사용자에게 명시하고 다음 작업을 안내하라.
