### 수정 4 (커밋 5d4cf3e): Worker 실패 로깅 + UI repaint yield
- Worker onerror에 console.error/warn 추가 (원인 파악용)
- sync fallback 진입 전 double-rAF yield → 제목/힌트 메시지 화면에 반영
- Worker URL을 new URL(…, location.href).href 절대경로로 변경

### 핵심 발견: Chrome file:// SecurityError
- 콘솔 에러: "SecurityError: Failed to construct 'Worker': Script at
  'file:///…/src/enum-worker-bundle.js' cannot be accessed from origin 'null'."
- 원인: file:// 로 열면 origin = null, Chrome이 다른 경로의 Worker 파일 로드 차단
- 이전 동작: 모든 Worker 실패 → sync fallback 60s 블로킹 → 결과 523개

### 수정 5 (커밋 7615f8f): Blob URL Worker — 근본 해결
- scripts/build-worker-bundle.js: enum-worker-init.js 자동 생성
  - 번들 전체를 JSON.stringify로 인라인 문자열화
  - Blob URL 사전 생성 → window._enumWorkerBlobUrl 등록
- battery_pack_renderer.html: enum-worker-init.js 스크립트 태그 추가 (app-render.js 직전)
- app-render.js: window._enumWorkerBlobUrl 우선 사용, 없으면 파일 URL 폴백
- 결과: 22 Workers 병렬, UI 비블로킹, **4612개 후보** (sync fallback 523개 대비 8.8배)
