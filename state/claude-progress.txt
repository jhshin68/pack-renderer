## 세션 11 메모 (2026-04-17 — 세션 10 UI 이슈 근본 해결)

### ✅ 해결 완료 — 세션 10 미해결 UI 레이아웃 이슈

**근본 원인**: Playwright 브라우저의 "물리 창 크기 ≠ CSS 뷰포트" 분리.
세션 10 CSS 6회 시도가 모두 Playwright 뷰에서만 어긋나 보였을 뿐,
실제 Chrome에서는 문제없이 작동. 사용자가 Playwright 창을 수동 드래그해도
CSS의 `100vw`/`100vh`는 Playwright가 고정한 값(1280×720 등)만 참조하므로
앱이 창 좌상단 일부에만 그려지고 나머지는 body bg만 보이는 증상이 발생.

### 적용한 CSS 수정 (v0.2.24 → v0.2.25)

1. html, body { height: 100% } 확정 추가
2. body: height:100vh → height:100% (정합성)
3. .app: height:100vh + width:100vw → position:fixed; inset:0
   · 어떤 환경에서도 뷰포트 전체를 확실히 덮음
4. .sidebar / .main / .right-panel: height:100vh → height:100%
5. grid-template-columns: 260px 1fr 280px → clamp(260px,20%,380px) 1fr clamp(280px,21%,400px)
   · 사이드바가 브라우저 폭에 비례해 확장 (min 260·280 / max 380·400)
6. 캐시 무효화 메타 3종 추가 (Cache-Control, Pragma, Expires)
7. 사이드바 폰트 전체 +20~30%
   · ctrl-label 8.5→10.5, seg-btn 9→11, num-val 11→13, toggle-label 10→12
   · hint 9→11, info-box 9→11, info-title 8→10, theory-tag 8→9.5
   · rp-title 11→13, cand-name 10→12, cand-scores 9→11, cand-desc 8→10

### app.js 변경

- fixSVGSize() 수정 시도했으나 유저가 이전 상태 선호 → 원복 완료
  (픽셀 고정 + max-width:100% 유지)

### 교훈 — Playwright 브라우저 사용 금칙

- UI 레이아웃 검증은 반드시 사용자의 실제 Chrome/Edge에서 확인
- Playwright의 자동 측정값이 숫자상 정상이어도 실유저 경험과 다를 수 있음
- CSS 변경 후 "Playwright에서 동작 확인" ≠ "실제 브라우저에서 동작 확인"
