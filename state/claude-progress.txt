## 세션 55 핵심 성과 — 비연속 그룹 고정 탐색 (Sparse Pinned Groups)

### 기능 개요
기존 pinned_groups는 G0, G1, ... 연속 순서로만 고정 가능.
이번 세션에서 임의 인덱스 고정(sparse)을 구현:
  예) G0 고정 + G11·G12 고정, G1~G10만 DFS 탐색

### 입력 형식
  연속: 줄당 1그룹 (기존 동작 유지)
  Sparse: "그룹번호: 셀목록" 형식
    0: r3c0 r2c0 r1c0 r2c1
    11: r3c10 r2c11 r1c10 r0c8
    12: r3c11 r2c12 r1c11 r0c9
  한 줄 입력도 지원: "0: ... 11: ... 12: ..." (자동 분리)

### 커밋 이력
- 847caf9: feat(sparse-pinned) — 핵심 알고리즘 구현 (TDD 6/6)
  - gen-enum.js: sparsePinnedMap, useSparsePin(), dfsCustom sparse 체크
  - app-state.js: parsePinnedGroups() → {mode, groups|sparseGroups}
  - app-render.js: sparse 분기, pinnedCellIdxGroupsSparse 워커 전달
  - enum-worker.js: pinnedCellIdxGroupsSparse 수신·변환
  - battery_pack_renderer.html: 도움말 sparse 형식 추가

  - 원인: "0: ... 11: ... 12: ..."가 한 줄로 입력되면
    11:·12: 패턴을 G0 셀로 모두 귀속 → G0만 12셀로 인식
  - 수정: split(/(?=\s\d+\s*:)/) lookahead로 인라인 그룹 분리
  - app-render.js: sparse 셀 매칭 실패 시 콘솔 경고 + UI 워닝 추가

    → 첫 번째 자유 그룹(Gk) 후보 수집 + sparse_first_free_idx 반환
  - app-render.js: isSparse guard 제거, sparseFirstFreeIdx 수집·전달
  - enum-worker.js: sparse + gk 조합 → pinned_groups_sparse 확장 전달
  - 결과: G0 sparse 핀 시 G1 후보 N개 병렬 분할 탐색

### 동작 원리 (G0+G11+G12 핀 예시, S=13)
1. G1 후보 목록 수집 (enumerate_g0_only + sparse)
2. G1 후보 수만큼 워커에 분배
3. 각 워커: pinned_groups_sparse = {G0, G1(고정), G11, G12}로
   G2~G10만 DFS → G11·G12 인접 검증 후 commitCandidate
