# 셀 홀더 배열 제약 설계 — 전문가 검토 메모

> 목적: 13S3P와 같은 논리 구성(S·P)이 주어져도 홀더 배열(rows×cols+빈슬롯)과 셀 배치 경우의 수가 폭증하는 문제를 해결하기 위한 제약 설계 방향 정리.
> 작성일: 2026-04-17
> 대상: Pack-Renderer LAYER 1·3 확장 검토

---

## 1. 문제 정의

### 1.1 현재 한계
- S·P가 고정되어도 물리적 홀더 배열은 유일하지 않다.
  - 예: 13S3P (N=39) → 13×3, 7×6(+3 빈슬롯), 커스텀 rows[] 등 다수 가능
- 홀더 배열이 확정되어도 셀 번호/그룹 배치 경우의 수는 여전히 수만 개 규모
- 현재 입력 스키마는 `rows[]` 한 필드에 "홀더 모양 + 빈 슬롯 + 논리 구성"이 암묵적으로 섞여 있어 모호

### 1.2 해결 요구사항
- 경우의 수를 실용적 범위(수십 개 이하)로 축소할 제약 체계
- 제약을 **계층화**하여 사용자 입력 부담을 최소화
- 기존 26개 원칙(LAYER 0·1·2)과 정합

---

## 2. 결론

**홀더 배열(rows×cols+빈슬롯)과 논리 구성(S·P)을 분리**한 뒤,
**"B+/B− 방향을 1차 제약, 시작/종료 셀 위치를 fallback"으로 계층화**하고,
**SFMT Step C에 Hamiltonian path 열거기를 내장**하는 3단 구조가 현재 프레임워크와 가장 잘 맞는다.

---

## 3. 핵심 근거

### 3.1 LAYER 1은 이미 "홀더 배열 ≠ S·P"를 허용하고 있다

`skills/nickel_plate_principles.md` 원칙 8은 다음을 명시한다.
- ① 행별 셀 수 r1, r2, r3…가 달라질 수 있다
- ② 홀더 슬롯 수가 N보다 많으면 빈 슬롯이 생긴다
- ④ grid_cols × grid_rows 방식은 직사각형 정배열의 특수 케이스일 뿐이다

즉 7×6 홀더에 13S3P(39셀+3빈슬롯)는 원칙 위반이 아니다.
다만 현재 입력 스키마는 `rows[]` 한 필드로 홀더 모양·빈 슬롯·논리 구성을 섞어서 받고 있어, 검증·UI·algorithm 레이어에서 모호함이 남는다. **입력 필드 분리가 선결 과제다.**

### 3.2 제약 우선순위: B+/B− 방향 ≫ 시작/종료 셀 위치

| 제약 | 성격 | 근거 | 실무성 |
|---|---|---|---|
| **B+/B− 출력 방향** | 경계 제약(soft) | 원칙 4·22가 이미 이 입력 전제 | 제품 사양(단자 위치·BMS 면)과 직결, 가장 자연스러움 |
| **시작/종료 셀 슬롯** | 노드 고정(hard) | Hamiltonian path 양 끝단 특정 | 중앙 출력·비대칭 케이스 전용 fallback |

- B+/B− 방향이 주어지면 경계 셀 집합에서 시작/종료 후보가 자동 유도된다.
- 두 제약을 **병렬**로 두면 입력 충돌 가능성이 생긴다. **계층화**가 옳다.

### 3.3 홀더 배열 확정 후 문제의 본질 = Hamiltonian Path 열거

`rows×cols` 홀더 위에서:
- P개 인접셀이 하나의 그룹(원칙 21 조건 A)
- S개 그룹이 직렬 체인(원칙 26)으로 이어짐
- 각 그룹 경계에서 인접 셀 쌍 ≥ 1개(원칙 21 조건 B)

이는 정확히 **"grid 위에서 S-segment Hamiltonian path 열거"** 문제다.
제약 없이 13S3P는 수만 개 해가 나오지만:
- **B+/B− 경계면 고정** → 시작·종료 엣지 집합 축소
- **ICC 5제약(원칙 16) Step C 하드 가지치기** → 행 스팬·종횡비·볼록성·비중첩·대칭 위반 즉시 컷
- **원칙 21 품질 점수** → −10점 후보 우선 가지치기

위 3단 결합으로 실용적 범위(수십 개 이하)로 축소 가능하다.

### 3.4 빈 슬롯은 엣지 배치가 정답

| 관점 | 내부 배치 | 엣지 배치 |
|---|---|---|
| 열 경로 | 홀더 중앙 공동 → 국부 핫스팟 | 대류·복사 균질 |
| 기구 강성 | 중앙 관통 공동 → 굽힘 강도 저하 | 외곽 절삭과 유사, 영향 미미 |
| 니켈 planar(원칙 16 ④) | 빈 슬롯 우회로 → 교차 위험 증가 | 경계선 단순, 교차 리스크 낮음 |
| 인접성(원칙 9) | 내부 끊김 → 그룹 분할 유인 | 외곽 끊김만 → 영향 국소 |

→ 사용자 수동 지정 허용하되, 자동 배치 기본값은 **"경계 우선"** 휴리스틱.

### 3.5 커스텀 배열에도 동일 파이프라인 작동

- 원칙 9 ③항: 거리 ≤ pitch × 1.1로 인접성 정의됨
- 원칙 24 ③항: 커스텀은 BFS로 그룹 번호 부여
- → Hamiltonian path + B+/B− 경계 제약 + BFS 번호 부여로 정배열·엇배열·커스텀 **공통 파이프라인** 성립

## 4. 제약 계층 설계안

### Level 1 — 필수 입력
```yaml
holder_shape:
  pattern: square | staggered | custom     # 원칙 17
  rows: <int>                              # 홀더 행 수 (물리)
  cols: <int>                              # 홀더 열 수 (물리)
  empty_slots: [[row,col], ...]            # 빈 슬롯 좌표 (선택)
logical_config:
  S: <int>                                 # 직렬 그룹 수
  P: <int>                                 # 그룹당 병렬 셀 수
```
**검증식**:
- `N = S × P`
- `rows × cols − |empty_slots| ≥ N`
- `pattern == "custom"`이면 `rows[]` 배열로 대체 입력

### Level 2 — 권장 입력 (경우의 수 1차 축소)
```yaml
terminal_direction:
  b_plus_side: top | bottom | left | right      # 원칙 4
  b_minus_side: top | bottom | left | right     # 원칙 4
bms_side: top | bottom | left | right           # 원칙 22 (기본: top)
```

### Level 3 — 선택 입력 (Fallback)
```yaml
path_hint:
  start_cell: [row, col]                   # 원칙 24 시작 셀 고정
  end_cell: [row, col]                     # 원칙 24 종료 셀 고정
  path_shape: straight | U | S | custom    # 원칙 4 경로 타입
```

    ↓

## 7. 미정·추후 검토 사항

- **Hamiltonian path 열거기 복잡도**: grid 크기에 따라 NP-hard. 10×10 이내는 실용 가능 추정, 이를 초과할 경우 휴리스틱·빔 서치 도입 필요.
- **빈 슬롯 자동 배치 휴리스틱**: 경계 우선이 기본값. 열 해석 결과와 연동한 가중치 모델은 v8에서 검토.
- **중앙 출력(BMS가 팩 중앙)**: 현재 `bms_side ∈ {top, bottom, left, right}`만 지원. 중앙 출력 요구 시 Level 3 `start_cell` 입력으로 우회.
- **커스텀 배열에서의 ICC 재해석**: 원칙 16 엇배열 재해석 규칙과 유사한 "커스텀 전용 재해석 규칙"이 필요할 수 있음. v8 과제.
