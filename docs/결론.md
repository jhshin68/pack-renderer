결론

맞습니다. 지금 단계의 병목은 **코드 품질이 아니라 설계 모델의
부정확성**입니다.\
따라서 다음 세션의 1순위는 알고리즘 개선이 아니라, **5P 병렬부를
"등임피던스 분산 네트워크"로 정의하는 설계 규격을 먼저 고정하는
것**입니다.\
이 규격이 고정되기 전에는 어떤 코드도 안정적인 설계 엔진이 되기
어렵습니다.

핵심 근거

지금까지 정리된 내용에서 가장 중요한 판단은 3가지입니다.

첫째, **5P 병렬부는 배선 문제가 아니라 전류 분배 구조 문제**입니다.\
즉 "모든 셀이 동일 전류를 갖도록 하는가"가 본질이고, 이를 위해서는 각
셀에서 trunk까지의 경로가 단순히 가깝거나 멀다는 수준이 아니라, **전체
경로 임피던스가 균등**해야 합니다.\
그래서 1x5 직선형이나 단일 feed 구조는 레이아웃이 단순해도 전류 편중이
커질 수밖에 없고, **양방향 공급과 분산 trunk를 갖는 구조가 기본 해법**이
됩니다.

둘째, **기존 구현 실패는 탐색 알고리즘 문제가 아니라 해 공간 정의가
틀렸기 때문**입니다.\
Greedy가 실패한 이유는 탐색 성능 부족보다,\
"좋은 구조"를 조합적으로 찾으려 했기 때문입니다.\
하지만 실제로는 5P 병렬부는 자유도가 큰 배선 조합이 아니라,
**전기적·기계적·제조적 제약을 동시에 만족하는 소수의 유효 토폴로지
집합**으로 보는 편이 맞습니다.\
즉, 5P는 탐색 대상이 아니라 **표준 블록**이어야 합니다.

셋째, **니켈은 선분이 아니라 제조되는 금속 구조물**입니다.\
이 점이 매우 중요합니다.\
기존 모델이 길이 중심, 거리 중심, 그래프 edge 중심으로 축소되면서 실제
제조 현실이 빠졌습니다.\
그러나 실제 니켈 구조는 trunk, branch, neck, 접합부 폭 변화, 용접점
위치, 열영향, 퓨즈 역할까지 포함하는 **geometry 기반 구조물**입니다.\
따라서 설계 엔진도 선 연결 생성기가 아니라, **제조 가능한 geometry
생성기**로 바뀌어야 합니다.

실행항목

1.  **5P 병렬 표준 규격서를 먼저 확정**\
    아래 4가지를 문서로 고정해야 합니다.

    - 허용 구조: H형, dual feed, offset stagger

    - 금지 구조: 1x5 직선형, single feed, 비대칭 branch

    - 등임피던스 판정 기준: 경로 길이만이 아니라 폭·두께·공유 trunk 반영

    - 제조 제약: 최소 neck 폭, 용접 금지 구간, 니켈 겹침, 조립 가능성

2.  **5P를 조합 문제가 아니라 블록 라이브러리로 정의**\
    예를 들면 아래처럼 가야 합니다.

    - Type-A: 2-3 stagger + dual feed

    - Type-B: 2-2-1 stagger + central H trunk

    - Type-C: offset mirrored pair structure\
      각 타입마다\
      셀 좌표, trunk 형상, branch 수, feed 위치, fuse neck 위치, 금지
      배치까지 포함한\
      **완성형 템플릿**으로 만들어야 합니다.

3.  **그 다음에만 13S5P 전체를 pack-level 배치 문제로 전환**\
    이 단계부터는 비로소 코드가 의미가 있습니다.\
    문제는 "셀 65개를 어떻게 잇나"가 아니라\
    \*\*"정의된 5P 블록 13개를 주어진 공간에 어떻게 놓고 직렬 연결까지
    완성하나"\*\*가 됩니다.\
    즉 순서는\
    **parallel block definition → block placement → series routing →
    manufacturable nickel geometry generation**\
    으로 가야 합니다.

한 줄로 다시 정리하면,

**다음 세션의 목표는 알고리즘 수정이 아니라, 5P 병렬 구조의
전기·기구·제조 기준을 고정한 "표준 블록 규격"을 먼저 만드는 것**입니다.\
그 규격이 확정되면 이후 코드는 훨씬 단순해지고, 그때부터는 탐색 코드가
아니라 실제 설계 엔진으로 발전할 수 있습니다.

결론

다음 단계는 바로 \*\*"5P 병렬 표준 블록 규격을 하나로 확정"\*\*입니다.\
여기서는 선택지를 열어두지 않고, **실제로 쓸 수 있는 단일 기준
구조(Type-A)**를 고정하는 것이 맞습니다.

핵심 근거

5P 구조에서 가장 안정적인 해는 이미 명확합니다.\
전류 균등성, 제조성, 확장성까지 고려하면 최적 구조는 다음 조건을 동시에
만족해야 합니다.

- 양방향 전류 공급 (dual feed)

- 중심 분산 (centroid 기반)

- branch 길이 보상 가능

- 니켈 가공/용접 현실 반영

이 조건을 동시에 만족하는 구조는 사실상 하나로 수렴됩니다.\
즉, 탐색할 문제가 아니라 **정의해야 할 구조**입니다.

실행항목 (Type-A 표준 규격 확정)

1.  **셀 배치 구조 (고정)**

- 5P = 2-3 stagger 구조

- 중앙 3셀 + 양쪽 2셀 비대칭 배치

- centroid 기준으로 trunk 배치

배열 예:

● ●

● ● ●

2.  **니켈 구조 (핵심)**

- 중앙 horizontal trunk 1개

- 좌우에서 각각 feed (dual feed)

- 각 셀로 branch 분기

구조 개념:

Feed → ===== Trunk ===== ← Feed

│ │ │ │ │

● ● ● ● ●

3.  **등임피던스 구현 규칙**

- 중앙 셀 → 가장 짧은 경로 → branch 폭 ↓ (저항 증가)

- 외곽 셀 → 가장 긴 경로 → branch 폭 ↑ (저항 보상)

- 모든 셀의 (경로 저항 합) 동일화

즉,

- 거리 보상 = 폭/두께로 조정

- 절대 "같은 폭" 금지

4.  **퓨즈 넥 (제조 포함)**

- 각 셀 직전 neck 삽입

- 위치: branch 말단 (셀 접점 바로 앞)

- 역할:

  - 셀 단락 보호

  - 미세 저항 보정 요소

5.  **절대 금지 규칙 (명시적으로 고정)**

- 1x5 직선형

- 단일 feed 구조

- 중앙 trunk 없이 star형 분기

- branch 길이 보상 없는 동일 폭 구조

결론

이제 다음 단계는 명확합니다.\
**"Type-A 5P 블록을 기반으로 13S 전체 팩을 어떻게 배치할 것인가"를 단일
방식으로 확정**해야 합니다.\
즉, 문제는 더 이상 병렬이 아니라 **공간 배치 + 직렬 연결 구조

핵심 근거

5P 내부는 이미 등임피던스 구조로 고정되었기 때문에 변수에서 제외됩니다.\
따라서 남는 설계 자유도는 3가지뿐입니다.

- 13개 블록의 공간 배열 방식

- 블록 간 직렬 연결 경로

- 니켈의 레이어/교차 처리

이 단계에서 가장 중요한 것은\
**직렬 연결이 병렬 구조를 깨지 않도록 하는 것**입니다.

즉,

- 직렬 니켈이 특정 셀 쪽으로 치우치면 → 병렬 불균형 재발

- 따라서 직렬 연결도 반드시 **centroid 기준으로 연결**되어야 합니다

결국 pack 설계의 핵심 원리는 이렇게 정리됩니다:

👉 "병렬은 등임피던스, 직렬은 centroid 연결"

실행항목

1.  **13개 블록 배열 방식 고정 (Snake 구조)**\
    가장 안정적인 구조는 2열 Snake 배치입니다.

예:

\[1\] → \[2\] → \[3\] → \[4\] → \[5\] → \[6\] → \[7\]

↓

\[13\] ← \[12\] ← \[11\] ← \[10\] ← \[9\] ← \[8\]

이 구조의 장점:

- 직렬 경로 최소화

- 니켈 교차 최소

- 열 분산 균일

2.  **직렬 연결 위치 규칙 (Centroid Rule)**\
    각 5P 블록의 연결 기준은 반드시:

- trunk 중심점 (centroid)

- 또는 dual feed 중간 지점

절대 금지:

- 외곽 셀 직접 연결

- 한쪽 branch만 통해 연결

즉 구조:

\[Block A trunk center\] → \[Block B trunk center\]

3.  **니켈 구조 레이어 분리**

- 병렬 구조: 하부 레이어 (고정)

- 직렬 연결: 상부 레이어 또는 별도 스트립

이유:

- 병렬 등임피던스 유지

- 직렬 수정 가능성 확보

- 제조 공정 단순화

현재:\
→ 정해진 블록을 "어디에 놓고 어떻게 이어붙일까"

결론

즉, 개념 설계(블록/배치)는 끝났고, **실제 생산 가능한 형상으로 변환하는
것이 핵심**입니다.

핵심 근거

지금까지의 설계는 전기적으로는 완성되었습니다.\
하지만 실제 양산에서는 아래 조건을 만족하지 않으면 전부 무의미해집니다.

- 프레스/레이저 가공 가능 형상

- 용접 접근성 확보

- 열변형 및 뒤틀림 방지

- 조립 공정 순서 가능성

특히 중요한 점은\
\*\*니켈은 회로가 아니라 "금속 부품"\*\*이라는 것입니다.

따라서 설계의 최종 판단 기준은:\
👉 "전류가 잘 흐르는가"가 아니라\
👉 **"이 형상이 실제로 찍히고, 붙고, 조립되는가"**입니다.

실행항목

1.  **Trunk / Branch / Neck Geometry 수치화 (고정 규격화)**

반드시 수치로 고정해야 합니다.

- trunk 폭: 기준 전류 대비 온도상승 10\~15℃ 이하

- branch 폭:

  - 중앙 셀: 기준 대비 0.7\~0.8배

  - 외곽 셀: 1.2\~1.4배

- fuse neck:

  - 폭: 0.8\~1.5 mm (셀 용량/전류에 따라)

  - 길이: 최소 2\~3 mm (안정 절단 확보)

즉,\
👉 등임피던스는 "길이"가 아니라 **폭으로 보정**하는 구조로 고정

------------------------------------------------------------------------

2.  **용접 포인트 및 금지 영역 정의**

각 셀당 반드시 규정:

- 용접점 개수: 2\~4점

- 위치: neck 이후 (셀과 직접 접촉부)

- trunk에는 직접 용접 금지 (열 집중 방지)

추가로:

- 니켈 겹침 영역 → 용접 금지

- 직렬 니켈과 병렬 니켈 교차부 → 절연 or 높이 분리

------------------------------------------------------------------------

3.  **단일 피스 vs 분할 구조 결정 (중요)**

여기서 반드시 하나로 고정해야 합니다.

권장 결론:\
👉 **5P는 "단일 프레스 피스"로 제작**

이유:

- 위치 정렬 정확도 확보

- 공정 단순화

- 저항 편차 최소화

단, 13S 전체는:\
👉 **블록 단위 + 직렬 스트립 분리 구조**

------------------------------------------------------------------------

2.  13S = Snake 배치 (완료)

3.  직렬 연결 = centroid 기반 (완료)
