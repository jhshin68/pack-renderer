### 원칙 8 — 셀 수 불변

N = S×P 는 설계 확정 시 불변이다. 각 그룹 Gi는 항상 P개 셀을 가진다.

① 홀더/케이스 형상에 따라 행별 셀 수(r1, r2, r3...)가 달라질 수 있다
② 홀더 슬롯 수가 N보다 많으면 빈 슬롯이 생긴다
③ 시각 배치(행별 셀 수)가 달라져도 N과 각 그룹의 P는 변하지 않는다
④ grid_cols × grid_rows 방식은 직사각형 정배열의 특수 케이스일 뿐이다

**구현 금지**: 특정 S/P 값 하드코딩 금지. 10S3P, 13S4P 등 특정 조합을 상정한 고정값 사용 금지.

### 원칙 9 — 니켈 연결 인접성 절대 규칙

니켈 플레이트는 바로 인접한 셀끼리만 연결한다. 점프(셀 건너뜀) 절대 금지.

① 정배열: 상·하·좌·우 4방향 인접만 허용
② 엇배열: 상·하·좌·우 + hex 대각선 6방향 인접 허용
③ 커스텀: 두 셀 중심 간 거리 ≤ pitch × 1.5인 셀끼리만 인접으로 판정.
   (계수 1.5는 row_offsets에 의한 대각선 인접 √2·pitch ≈ 1.414를 허용하기 위한 값.
    문서 기준 1.1에서 코드 구현 1.5로 확장됨 — 대각선 인접 허용 목적)
④ 위반 시: 비인접 그룹 간 단락 또는 특정 셀 과부하 → 열폭주

### 원칙 26 — 직렬 경로 단일성 불변

그룹 간 직렬 연결은 단순 체인(simple path) 구조여야 한다. G0–G1–G2–…–G_{S-1} 순서로만 연결되며, 분기(branch)·루프(loop)·우회 경로는 금지한다.

① 중간 그룹 Gi (0 < i < S-1)는 정확히 G_{i-1}과 G_{i+1} 두 이웃만 가진다.
② G0와 G_{S-1}은 각각 직렬 이웃 1개만 가진다.
③ 이 제약은 그룹 레벨 토폴로지에만 적용된다. 그룹 내부 셀 간 병렬 연결에는 적용하지 않는다.
④ 위반 시: 전류 분기 → 일부 경로 미사용 → 니켈 형상 다의성 발생.

---

### 원칙 28 — 니켈플레이트 셀 수 불변 (P / 2P / P 구조)

S×P 팩에서 니켈플레이트의 총 수는 정확히 **S+1개**이며, 각 플레이트의 셀 수는 다음 구조를 반드시 따른다.

```
플레이트 #1       : P개 셀 (단독 집전, B+ 터미널)
플레이트 #2 ~ #S  : 2P개 셀 (인접 두 직렬 그룹을 동시 집전, 브리지)
플레이트 #(S+1)   : P개 셀 (단독 집전, B- 터미널)
```

① **총 수 불변**: 니켈플레이트 수 = S+1
② **셀 수 불변**: 첫·마지막 = P개, 중간 = 2P개
③ **연결성 불변**: 각 플레이트의 모든 셀이 BFS 기준 단일 연속 영역을 이뤄야 한다
④ **위반 시**: 단절 플레이트는 물리 제작 불가 → 설계 즉시 폐기

---

### 원칙 29 — 그룹 내 병렬 극성 일관성 불변 ★ (2026-04-19 신설)

그룹 Gk (k = 0, 1, …, S−1 의 **모든** 그룹)에 속한 P개의 셀은 동일 면(표면·이면)에서 **예외 없이 동일한 극성**을 가져야 한다.

**예시 (10S4P — G1·G2·…·G10 각 그룹에 동일 규칙 적용, G2로 예시):**

| 면 | 허용 여부 | 극성 패턴 | 이유 |
|---|---|---|---|
| 표면 (++++), 이면 (−−−−) | ✅ 허용 | 단극성(monoparity) | 정상 병렬 집전 |
| 표면 (−−−−), 이면 (++++) | ✅ 허용 | 단극성(monoparity) | 정상 병렬 집전 |
| 표면 (++−−), 이면 (++−−) | ❌ 금지 | 혼합 극성 | 동일 판 내 단락 |
| 표면 (+−++), 이면 (+−−−) | ❌ 금지 | 혼합 극성 | 동일 판 내 단락 |
| 표면 (−+++), 이면 (+−−−) | ❌ 금지 | 혼합 극성 | 동일 판 내 단락 |

**근거:** 병렬 연결된 셀들은 단일 집전판(니켈플레이트)으로 묶인다. 동일 면의 집전판이 이종 극성 셀을 동시에 접촉하면 판 내부에서 단락이 발생한다.

① **그룹 배정 제약**: `enumerateGroupAssignments()`는 각 그룹이 단일 극성 면(monoparity face)을 갖도록 셀을 배정해야 한다. 셀 배정 후 같은 그룹 안에서 표면 극성이 혼재하면 해당 후보는 폐기한다.

② **렌더러 구현 제약**: 렌더러는 셀 인덱스·열거 순번이 아닌 **그룹 배정(groupCells)** 을 직접 참조하여 극성을 결정해야 한다. 순번 기반 추정은 이 원칙을 위반한다.
   ```javascript
   // ✅ 올바른 구현 (cellGrpIdx 패턴)
   const cellGrpIdx = new Map();
   groupCells.forEach((gc, gIdx) => gc.forEach(c => cellGrpIdx.set(`${c.row},${c.col}`, gIdx)));
   const topCells = pts.map(pt => {
     const g = cellGrpIdx.get(`${pt.row},${pt.col}`) ?? 0;
     return drawCell(pt.x, pt.y, R, getCellPolarity(g, 'top'), params.scale);
   });

   // ❌ 금지 패턴 (snake 순번 기반 — 원칙 29 위반)
   const topCells = snake.map((pt, i) => {
     const g = Math.floor(i / cellsPerGroup); // ← 위반
     return drawCell(pt.x, pt.y, R, getCellPolarity(g, 'top'), params.scale);
   });
   ```

③ **위반 시**: 동일 니켈플레이트가 +/− 양극을 동시 단락 → 열폭주 → 설계 즉시 폐기.

### 원칙 4 — B+·B− 출력 방향이 셀 배열을 결정한다

B+·B−의 출력 위치(방향)가 먼저 결정되고, 그에 맞게 직렬 경로를 설계한 후, 그 경로를 따라 셀 면 방향(+/−)이 교번 배치된다.

① 직렬 경로는 좌→우 직선뿐 아니라 U자, S자, 임의 방향 모두 가능
② B+와 B−가 같은 방향에서 나와야 하면 U자형 경로로 설계
③ 셀 면 방향(+면/−면)은 직렬 경로를 따라가며 그룹마다 교번 적용
④ **2단계 방향 결정 구조**:
   - **1차**: `b_plus_side` / `b_minus_side` ∈ {top, bottom, left, right}
   - **2차**: `calcBoundarySet(cells, side)` 로 경계 셀 집합 도출

---

### 원칙 17 — 배열 선택 자유도 불변

배열 방식은 3-way 선택이다.

① 정배열(square): S×P 직사각 격자
② 엇배열(staggered): S×P 육각 격자
③ 커스텀(custom): 행별 셀 수 rows=[n1,n2,…] 수동 지정, S 별도 설정

**홀더 3-레벨 계층:**

| 레벨 | 파라미터 | 의미 | 구현 함수 |
|---|---|---|---|
| Level 1 | `holder_rows` × `holder_cols` | 물리 홀더 격자 크기 | `buildHolderGrid()` |
| Level 2 | `b_plus_side` / `b_minus_side` | B+/B− 출력 방향 | `calcBoundarySet()` |
| Level 3 | — (자동) | Hamiltonian 그룹 배정 열거 | `enumerateGroupAssignments()` |

### 원칙 14 — 면별 병합 불변

니켈판은 면별로 병합된다.

① 상면 플레이트: [G0] | [G1∪G2] | [G3∪G4] | …
② 하면 플레이트: [G0∪G1] | [G2∪G3] | …
③ B+는 기본값으로 상면 G0. 원칙 4에 의해 하면 배치 가능.
④ B−: S짝수 → 상면 G_{S-1} / S홀수 → 하면 G_{S-1}
⑤ 브리지 플레이트 연결 기준점: 원칙 22(BMS 최단 거리)로 선택

### 원칙 16 — ICC 산업 실무 제약 불변

ICC(Industrial Compactness Constraint) 5대 제약:

① 행 스팬 ≤ 2 (엣지 수직 리본 예외)
② 종횡비 ≤ 2.0
③ 볼록성 ≥ 0.75
④ 동일 면 이종 극성 플레이트 XY 투영 비중첩, 에지 간 간격 ≥ δ_min (기본값: nickel_w / 2)
⑤ 합동 쌍 대칭 배치

### 원칙 15 — 최소 형상 자동 탐색 (SFMT)

SFMT(Symmetry-First Minimum-Type) 4단계 알고리즘:

| 단계 | 내용 | 구현 함수 |
|---|---|---|
| Step A | 팩 대칭군 탐지 (C2/C4/C6 회전 대칭) | — |
| Step B | 국소 합동 쌍 열거 | — |
| Step C' | Hamiltonian 그룹 배정 열거 | `enumerateGroupAssignments()` |
| Step C | ICC 제약 가지치기 + 쌍-우선 구성 | — |
| Step D | 종수 최소화 | — |

**Step C' 전략:**
- 표준 4종: 보스트로페돈 L→R / R→L, 열 우선 L→R / R→L (항상 즉시)
- 백트래킹 보완: N ≤ 18 소형팩 (max 300,000 iter, max_candidates=40)
- 정렬: B+/B− 경계 준수 → ICC 위반 수 ↑ → 품질 점수 ↓ → m_distinct ↑

### 원칙 18 — 니켈 형상 알고리즘

니켈 형상 = [인접 셀 쌍 연결선 stroke=2R, linecap=round] + [셀 원형 배경 r=R]

**Convex Hull 방식 절대 금지**: 일렬 배치 그룹에서 hull inflate 시 팩 전체 70% 덮는 거대 도형 발생.

stroke graph G=(V,E): 그룹 내 셀 노드 V, 원칙 9 인접 관계 엣지 E.
planar 검증: adjacency spanning tree 순서로 stroke 추가 → 사이클 없음 보장.

### 원칙 23 — 엇배열 hex 클러스터

엇배열 그룹 형상: hex 펜토미노(2-1-2 다이아몬드 등).
인접성: 6-이웃 (원칙 9).

### 원칙 24 — 그룹 번호 부여 규칙

① 정배열·엇배열 기본: 보스트로페돈 (홀수 행 좌→우, 짝수 행 우→좌)

③ 커스텀: 원칙 9 거리 기반 인접성 그래프 따라 BFS 순서
④ 완료 후 확인: G0 = B+ 단자 그룹, G_{S-1} = B− 단자 그룹
