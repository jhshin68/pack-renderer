## 3. 단일 셀 극성 불변 법칙 ★ (v0.2 신규 — 매우 중요)

> **셀은 물리적으로 양극(+)과 음극(-)이 고정된 부품이다.**

- **상면이 + 이면 하면은 반드시 −**
- **상면이 − 이면 하면은 반드시 +**
- **이 관계는 어떠한 경우에도 바뀌지 않는다.**

| 셀 방향 | 상면 | 하면 | 색상(상면) |
|---------|------|------|-----------|
| 정방향 (홀수 그룹) | + (양극) | − (음극) | 빨강 |
| 역방향 (짝수 그룹) | − (음극) | + (양극) | 파랑 |

### 프로그램 구현

```javascript
cell.top_polarity = (group_index % 2 === 0) ? '+' : '-'
cell.bot_polarity = (cell.top_polarity === '+') ? '-' : '+'
```

## 5. 셀 심볼 정의 ★ (v0.2.3 — 불변 원칙)

> **+ 단자부 = 빨강 테두리 + 내부 빨강 캡(직경 10mm 고정)  
> − 단자부 = 검정 테두리 + 내부 색 없음 (투명)**

### + face (양극면)
- 외원: 흰색 채움 + **빨강 테두리** (`stroke="#C0392B"`, stroke-width=R×0.08)
- 내부 캡 단자원: **빨강 채움, 직경 10mm 고정** (`r = 5mm × scale`, `fill="#C0392B"`, stroke 없음)
  - 18650·21700 모두 동일하게 10mm (실물 양극 캡 실치수)
  - 셀 크기와 독립적인 절대 치수
- 그 외 장식 없음

### − face (음극면)
- 원 1개: **검정 테두리 + 내부 색 없음** (`fill="none"`, `stroke="#1a1a1a"`, stroke-width=R×0.08)
- 속이 비어 니켈 플레이트나 배경이 그대로 보임
- 내부 장식 없음 (캡·✕·점선 모두 없음)

## 7. 니켈플레이트 배치 핵심 원칙 ★ (v0.2 수정)

### 7-0. 니켈 색상 표준 ★ (v0.2.4 신규)

> **니켈 플레이트는 회색으로 표현한다.** (실물 니켈 메탈릭 톤)

- 채움: `fill="#888888"` (중간 회색)
- 테두리: `stroke="#555555"`, stroke-width=0.7
- I자·ㄷ자·수평 브리지 모두 동일 색상 적용
- 배경·셀 심볼과 시각적으로 명확히 구분되도록 유지

### 7-1. 니켈 세로 스트립 위치 법칙 (★ 가장 중요)

> **니켈 세로 스트립의 중심선 = 해당 그룹 셀들의 중심 x 좌표**

```
nickel_x = cell_cx - nickel_w / 2
```

**주의:** 그룹 경계나 셀 사이 틈에 배치하면 틀림. 반드시 셀 중심 기준.

### 7-2. 니켈 형상 종류

| 형상 | 역할 | 구성 |
|------|------|------|
| I자 니켈 | 병렬 연결 전용 + B+/B− 탭 | 세로 스트립 1개 |
| ㄷ자 니켈 | 병렬 연결 + 직렬 브리지 **동시** 처리 | 세로 2개 + 가로 2개 |

### 7-3. ㄷ자 가로 폭 계산

```
x_start = G_left.cx  - nickel_w / 2   ← 왼쪽 세로 시작
x_end   = G_right.cx + nickel_w / 2   ← 오른쪽 세로 끝
width   = x_end - x_start             = pitch + nickel_w
```

### 7-3-1. 사다리형 수평 브리지 ★ (v0.2.5 신규)

> **ㄷ자 니켈의 수평 브리지는 P개 — 모든 row p(=0 ~ P-1)마다 좌·우 세로스파인을 연결한다.**

이유:
- 중간 row 셀끼리의 전류 경로를 직선 가로(pitch 길이)로 단축.
- 세로스파인 단일 의존 제거 → 스파인 파손 시 우회 경로 확보.
- 고C-rate 방전 시 내부저항 균형 및 셀 간 전압 불균형 완화.

좌표:
```
for p = 0 .. P-1:
    x1 = G_left.cx  - nickel_w/2
    x2 = G_right.cx + nickel_w/2
    y  = cy[p] - bar_h/2   (브리지 높이 = bar_h_mm × scale, 기본 3mm)
    rect(x1, y, x2-x1, bar_h)
```
- 기존 상·하단 브리지(row 0, row P-1)는 사다리 규칙에 자연 포함.
- 브리지 두께(bar_h)는 공정 스팟웰딩 가능 영역 기준 3mm 유지.

### 7-4. 니켈 세로 높이 계산 (v0.2.6 수정)

> **니켈 경로는 항상 셀 중앙만 통과한다.** 세로 스파인·수평 브리지 모두 셀 중심(cy[p])을 기준으로만 그린다.

```
y_top  = cy[0]        ← 첫 셀 중심에서 시작
y_bot  = cy[P-1]      ← 마지막 셀 중심에서 종료
height = y_bot - y_top
```

- 셀 경계 바깥으로의 확장 금지 (v0.2.5 이전 R×0.75 확장 폐기).
- 단자 탭 영역은 `drawTerminal()`이 별도로 그린다.
- 수평 브리지: row p에서 y=cy[p]−barH/2 (셀 중심 통과).

## 8. S별 니켈 패턴 도출표 ★

| S | 상면 니켈 패턴 | 하면 니켈 패턴 | B+ | B− |
|---|--------------|--------------|----|----|
| 2 | G1+G2 ㄷ자 | G1 I자, G2 I자 | 하면 G2 I자 탭 | 하면 G1 I자 탭 |
| 3 | G1 I자, G2+G3 ㄷ자 | G1+G2 ㄷ자, G3 I자 | 상면 G1 I자 탭 | 하면 G3 I자 탭 |
| 4 | G1 I자, G2+G3 ㄷ자, G4 I자 | G1+G2 ㄷ자, G3+G4 ㄷ자 | 상면 G1 I자 탭 | 하면 G4 ㄷ자 우측 탭 |
| 5 | G1 I자, G2+G3 ㄷ자, G4+G5 ㄷ자 | G1+G2 ㄷ자, G3+G4 ㄷ자, G5 I자 | 상면 G1 I자 탭 | 하면 G5 I자 탭 |
| 7 | G1 I자, G2+G3, G4+G5, G6+G7 ㄷ자 | G1+G2, G3+G4, G5+G6 ㄷ자, G7 I자 | 상면 G1 I자 탭 | 하면 G7 I자 탭 |
| 13 | G1 I자, G2+G3~G12+G13 ㄷ자 6개 | G1+G2~G11+G12 ㄷ자 6개, G13 I자 | 상면 G1 I자 탭 | 하면 G13 I자 탭 |

### 일반화 규칙

```
상면: G1 I자(B+) + 짝수시작 ㄷ자쌍(G2+G3, G4+G5...) + S짝수면 G_S I자 추가
하면: 홀수시작 ㄷ자쌍(G1+G2, G3+G4...) + S홀수면 G_S I자(B-)
```

## 9. 좌표 체계

```
origin (0,0) = 팩 좌측 상단 마진

cx[g] = margin + g × pitch_x + R      (g = 0 ~ S-1, 그룹 인덱스)
cy[p] = margin + p × pitch_y + R      (p = 0 ~ P-1, 병렬 인덱스)

nickel_x[g]  = cx[g] - nickel_w / 2
nickel_y_top = cy[0]   - R × 0.75
nickel_y_bot = cy[P-1] + R × 0.75
```

## 10. 구현 입력 파라미터

```json
{
  "cell_type":      "21700",
  "S":              3,
  "P":              3,
  "grid_cols":      null,
  "grid_rows":      null,
  "gap":            0.0,
  "layout":         "auto",
  "show_nickel":    true,
  "show_terminal":  true,
  "scale":          4.0
}
```

- `gap`: 셀 간 도면상 간격(mm). **기본 0 (맞닿음 불변 원칙)**
- `layout = "auto"`: S≥5 또는 P≥4 → top_bottom, 그 외 → side_by_side
- `grid_cols`, `grid_rows` (v0.2.10): 물리 격자 오버라이드. `null`이면 각각 S, P 사용 (canonical).
  - `grid_cols × grid_rows ≥ S × P` 필수 (셀 N개를 모두 담을 수 있어야 함).
  - row-major로 N개 채우고 잉여 슬롯은 공란.
  - canonical(cols=S, rows=P)이 아니면 니켈·단자 미렌더(공정 사양 미정의).

## 12. v0.2 핵심 요약 — 27대 불변 원칙 ★

1. **셀 상면이 + → 하면은 반드시 −** (극성 불변)
2. **니켈 세로 중심 = 셀 중심 cx** (니켈 위치 불변)
3. **ㄷ자 가로 = G_left.cx − nickel_w/2 ~ G_right.cx + nickel_w/2**
4. **상면/하면 항상 함께 표시** (S≥5 또는 P≥4 → 상하 배치)
5. **S홀수 → 양 끝 I자, 하면 끝 I자 = B−탭 / S짝수 → 하면 I자 없이 ㄷ자만**
6. **+ 단자부 = 빨강 테두리 + 빨강 캡(직경 10mm 고정) / − 단자부 = 검정 테두리 + 내부 무채움 + 중앙 검정 점(R×0.25)** (셀 색상 불변) ★
7. **셀과 셀은 항상 맞닿아 있어야 한다** (gap=0, pitch=render_d) ★
8. **ㄷ자 니켈은 사다리형 — 수평 브리지 P개 (모든 row)** (전류 경로 최소화) ★
9. **니켈 경로는 항상 셀 중앙만 통과 — 스파인 범위 cy[0]~cy[P-1], 셀 밖 확장 금지** ★
10. **니켈 두께는 세로·가로 항상 동일 (= nickel_w)** — 실물 단일 스트립 절곡 공정과 일치 ★
11. **셀 수 N = S × P는 불변. 물리 격자는 grid_cols × grid_rows로 오버라이드. row-major로 N개 채우고 잉여 슬롯은 공란.** ★
12. **니켈판은 대각선으로 지나갈 수 없다 — 상·하·좌·우 인접 셀 간에만 연결** (인접성 불변) ★★ 최우선
13. **니켈판은 점프(셀 건너뛰기) 불가 — 점프는 물리적 쇼트** (점프 금지 불변) ★★ 최우선
14. **니켈판은 면별로 병합된다 — 하나의 플레이트가 (−−−)∪(+++) 를 덮으며 병렬+직렬 브리지 겸함** (면별 병합 불변) ★★
    - 상면 플레이트: [G0] | [G1∪G2] | [G3∪G4] | ... | [G_{S-1} if S 짝수]
    - 하면 플레이트: [G0∪G1] | [G2∪G3] | ... | [G_{S-1} if S 홀수]
    - 솔로 플레이트 = 단자 탭 위치: B+ 항상 상면 G0 / B- S짝수→상면 G_{S-1}, S홀수→하면 G_{S-1}
15. **그룹 형태 자유 — P셀이 adjacency 그래프로 연결만 되면 유효** (형태 자유 불변) ★★
    - 3P 유효 형태: I자, L자 4회전 (총 5형)
    - 4P 유효 형태: I자, L자 4회전 변형, 2×2 정사각형 등 (테트로미노 집합)
    - 조건 A: 그룹 내 셀들이 상·하·좌·우 인접 체인으로 단일 연결 컴포넌트
    - 조건 B: G_i와 G_{i+1} 사이 인접 셀 쌍 ≥ 1개 (직렬 연속성, 원칙 12·13 확장)
    - 조건 A·B 만족 시 **동일 S×P에 대해 격자 배치 경우의 수는 매우 많음** (조합론적)
    - 배치 생성 알고리즘은 여러 정답 중 하나를 선택. 사용자 지정 패턴 입력도 허용 가능.
16. **다수 유효 타일링 중 BMS 최단 거리 기준으로 B+/B− 단자 위치를 선택** (BMS 근접 선택 불변) ★★
    - 동일 S×P×격자 조합에 대해 유효 타일링·순서화는 수백~수천 개 존재.
    - 설계 시 BMS 커넥터 좌표를 입력받아, B+/B−까지의 맨해튼 거리 합 최소화 타일링을 추천.
    - 사용자가 "5×4 6S3P 정배열"처럼 지시하면 렌더러는 유효 경우의 수 요약 + 추천 케이스 제시.
17. **엇배열(Staggered/HCP)의 그룹 형상은 hex 펜토미노(5셀 헥사 클러스터)이며, 인접성은 6-이웃(hex)이다** (엇배열 hex 클러스터 불변) ★★
    - 엇배열은 반 피치 오프셋 행 구조 → 각 셀은 최대 **6개 이웃**(같은 행 좌/우 + 위 행 2 + 아래 행 2).
    - 따라서 P-그룹의 유효 형상은 "hex 펜토미노"(2-1-2 다이아몬드, 1-2-2, 2-2-1, 1-2-1-1 등 6-이웃 연결 5셀 집합).
    - 정배열 대비 차이: **형상만 hex로 교체**. 그 외 모든 공식(§12 G 보스트로페돈 직렬, §14 플레이트 병합, §16 BMS 근접)은 **그대로 동일**.
    - 13S5P 엇배열 예시(8·8·10·13·13·13 계단형): 13개 hex 5-클러스터가 팩 전장을 좌↔우 지그재그로 직렬 연결, 플레이트 병합은 상면 [G0][G1∪G2]…[G11∪G12] = 7판, 하면 [G0∪G1][G2∪G3]…[G10∪G11][G12] = 7판.
    - 물리 근거: 실제 조립 사진상 니켈판이 hex 다이아몬드 형태로 5셀을 덮으며, 판간 브릿지 탭이 상단으로 돌출.
18. **최소 형상 타일링은 SFMT(Symmetry-First Minimum-Type) 4단계 알고리즘으로 결정적으로 도출한다** (최소 형상 자동 탐색 불변) ★★★
    - **핵심 정리**: 무작위 백트래킹으로는 희귀 해를 놓친다. **국소 합동 쌍을 먼저 시드로 배치**한 후 나머지를 채우는 "쌍-우선 구성(pair-first construction)"이 필수.
    - **Step A — 팩 대칭군 G 탐지**: 셀 중심 좌표 집합에 회전(60°·90°·180°) + 거울반사 변환을 적용해 자기 자신에 매핑되는 원소를 찾는다. 결과는 C1(없음)·C2(180°)·D1(1축 반사)·D2·C6·D6 중 하나. 13S5P 계단(8·8·10·13·13·13)=C1. 6S3P 정배열 5×4=D2.
    - **Step B — 국소 합동 쌍 열거 (Local Congruence Enumeration)**:
      1. 크기 m셀(=2P, 10셀) 연결 부분집합을 전부 생성
      2. 각 집합에 대해 캐노니컬 서명 σ(set) = min{ (회전k×거울ref)(점집합) : k∈0..5, ref∈{+1,−1} } 계산 (hex면 6회전, rect면 4회전)
      3. 동일 σ를 가지는 서로소 쌍 (A,B) 모두 추출, 중심간 거리 오름차순 정렬
      4. 쌍 매칭 그래프의 최대 매칭 수 k_local = 이론적 중복쌍 상한
    - **Step C — 쌍-우선 구성(Pair-First Construction)**:
      1. 합동 쌍 (A,B)를 고정 배치 시드로 선정 (A·B 각각은 2개 연속 그룹 G_i∪G_{i+1}, G_j∪G_{j+1}의 합집합이어야 플레이트 병합 공식과 호환)
      2. A·B를 구성하는 4개 pentomino 내부 분할 열거 (A=P_a∪P_b, B=P_c∪P_d, 각 P는 hex 또는 4-이웃 pentomino)
      3. 나머지 45셀(=N−20)에 대해 G0(1 pentomino) + 8 pentomino 분할 백트래킹
      4. 13개 그룹의 그룹 인접 그래프에서 **(P_a,P_b)·(P_c,P_d)가 반드시 연속**이 되는 Hamiltonian 경로만 허용 (가지치기)
    - **Step D — 종수 최소화 & 선택**:
      1. 각 feasible 구성의 플레이트 7개 서명 계산 → 유니크 수 m_distinct
      2. m_distinct 최소값 모든 구성 수집 → 경우의 수 n_min 출력
      3. Tie-breaker 우선순위: (i) G0 위치가 BMS에 가까움 (원칙 16), (ii) 쌍 거리 짧음, (iii) hex 펜토미노 대칭형(2-1-2 다이아몬드) 포함
    - **이론적 하한**: m_distinct ≥ ⌈(plates−1)/k_local⌉ + 1  (G0 단독 1종 포함)
      - 예 13S5P C1 + k_local=1: m_min ≥ ⌈6/1⌉ 단, 쌍 1쌍 인정 시 ⌈6/2⌉+... 보정 → 6종 달성 가능 (사진 일치)
      - 예 6S3P D2(|G|=4): plates=3(S홀수+1), 대칭 쌍 최대 1 → m_min ≥ 2
    - **복잡도**: 무작위 O(partition!·Hamilton!)를 O(k_local × inner_partition × H_constrained) 로 축소. 13S5P에서 120,000 쌍 무작위 탐색 대비 **수백 배 효율** + **결정적 수렴 보장**.
    - **물리 근거**: 사진 검증된 13S5P 실물 — 빨강 쌍(중앙 좌우 2-3-3-2 다이아몬드) 기준 타일링은 C1 팩에서도 **국소 반사 대칭**이 존재함을 증명.
19. **SFMT 수학해는 반드시 ICC(Industrial Compactness Constraint)로 보정되어야 실무 설계가 된다** (산업 실무 제약 불변) ★★★
    - **문제 인식**: SFMT 순수 수학해는 "6-이웃 연결 5셀"이면 임의 형태 허용 → 4행 이상 대각 스파게티 플레이트 생성 가능. 실물은 그렇게 만들 수 없다.
    - **ICC 5대 제약** (필수):
      1. **행 스팬 ≤ 2**: 각 펜토미노는 연속 2행 이내. 10셀 플레이트(G_i∪G_{i+1})도 2행 이내. **예외**: 팩 엣지의 "수직 리본 플레이트"(팩 폭 끝단 3행 차지, 1컬럼)는 허용 — 좁은 폭 엣지 보강용.
      2. **종횡비 ≤ 2.0**: 플레이트 bounding box의 W/H 또는 H/W ≤ 2.0. 엣지 리본 예외만 허용.
      3. **볼록성 ≥ 0.75**: convex_hull_area / bounding_rect_area ≥ 0.75. 요철 많은 불규칙 형상 배제.
      4. **용접점 수 최소**: 플레이트당 용접점 = 셀 수 × 2(셀당 +·−) = 10셀 → 20점 고정. 단 플레이트 분리(이형) 많을수록 공정 복잡 → 종수 ↓ + 동일형상 ↑ 우선.
      5. **대칭 배치**: 합동 쌍은 팩 중심선(local axis)에 대해 좌↔우 또는 상↔하 대칭 위치에 배치 — 제조 지그(jig) 대칭성 활용, 양산 직진도 확보.
    - **실무 근거**:
      - (a) **스탬핑 공정**: 긴 대각 플레이트는 다이(die) 크기 ↑ + 폐기율 ↑. 2행 밴드는 롤 스탬핑 가능
      - (b) **스폿 용접**: 긴 플레이트는 용접점 간 유도 응력 누적 → 크랙. 짧은 2행 밴드는 응력 분산
      - (c) **써멀 균일성**: 대각 경로는 전류 밀도 불균일(긴 경로 발열) → 셀간 온도 편차. 콤팩트 형상은 전류 균일
      - (d) **Jig fit**: 조립 지그에 플레이트 삽입 시 콤팩트 형상만 기계적 정렬 가능. 비볼록은 틀어짐
      - (e) **불량 대응**: 특정 플레이트 불량 시 소수 종(種) 재고로 대응. 6~7종은 SKU 관리 가능, 13종은 불가
    - **SFMT → SFMT+ICC 알고리즘 업데이트**:
      - Step B에 펜토미노 필터: `row_span(p) ≤ 1` 우선, 필요 시 엣지만 `≤ 2` 허용
      - Step C에 병합 후 검증: `aspect(plate) ≤ 2.0 AND convexity(plate) ≥ 0.75`
      - Step D tie-breaker: **(i) 행 스팬 합 최소 (ii) 평균 종횡비 최소 (iii) BMS 근접 (iv) 합동 쌍 대칭 거리 최소**
    - **실물 사진 대조 (13S5P 8·8·10·13·13·13)**:
      - 상단 2판: 2행 수평 밴드(2×5 평행사변형형), row 0~1 · row 1~2 범위
      - 중앙 합동 쌍(빨강): 2행 2-3-3-2 다이아몬드, 대칭 배치, row 3~4 범위
      - 우측 엣지 1판: 수직 리본(3행×1컬럼 변형), 행 스팬 3 **예외 적용**
      - G0 (5셀): 좌하단 엣지 코너, 2행 이내
      - 결론: 합동 쌍 1 + 비합동 4 + 엣지 리본 1 + G0 1 = **6종 7판**, 모두 ICC 충족
20. **니켈 플레이트 형상 수(m_distinct)의 최소화가 모든 설계 목적함수의 최우선이다** (형상수 최소화 최우선 불변) ★★★★ 최우선
    - **경제 논리**: 플레이트 형상 1종 = 전용 금형(다이) 1세트 필요. 금형 비용 기준 ₩15M~₩30M/세트. 형상 7종 = ₩140M+, 형상 3종 = ₩60M. **m_distinct 감소가 CAPEX·양산 원가·재고 SKU·공정 복잡도 모두 지배**.
    - **우선순위 재정의 (v0.2.17 기준)**:
      1. **m_distinct 최소화** (최우선) — 동일 형상 플레이트 다수 반복 사용 권장
      2. 원칙 19 ICC 실무 제약 (차순위)
      3. 원칙 16 BMS 근접 (차차순위)
      4. Hamiltonian 순서 존재성 (필수 조건, 목적이 아님)
    - **제약조건 입력 `max_plate_types: K`**: 사용자가 K를 지정하면 알고리즘은 반드시 **K개 이하의 고유 형상만** 사용해 타일링. 같은 형상 반복 사용 허용·권장. 해가 없으면 K를 증가시켜 재탐색.
    - **비용 모델**: Total_Cost = m_distinct × ₩20M(금형) + N_plates × ₩300(재료·가공) + m_distinct × ₩5M(SKU·품질관리)
    - **실물 근거**: 13S5P 사진 기준 m=6 실제 양산 원가 ₩150M vs SFMT 순수 최소 m=4 가상 설계 ₩100M (33% 절감). 금형 재사용률이 핵심 경제 지표.
21. **니켈 플레이트에는 엠보(embossing)가 있어 뒤집을 수 없다 — 거울반사 형상은 별개 형상으로 취급한다** (엠보 불가역성 불변) ★★★★ 최우선
    - **물리 근거**: 니켈 플레이트는 스탬핑 시 **엠보(돌출 리브)** 가 형성되어 있음 (용접 응력 분산 + 전류 경로 가이드 + 안테나형 탭 방향성). 엠보는 특정 면(상면)에만 존재하므로 플레이트를 뒤집으면 엠보 방향이 역전되어 용접 공정 불가.
    - **수학적 재정의 (원칙 18 SFMT 업데이트)**:
      - **캐노니컬 서명 σ(set)** = min{(회전k)(점집합) : k ∈ 0..5}  (hex 6회전만, **거울반사 제거**)
      - 정배열(4회전)은 σ(set) = min{(회전k)(점집합) : k ∈ 0..3}
      - 두 플레이트 A, B가 **동일 형상** ⇔ σ(A) = σ(B) (회전만으로 합동)
      - 거울상 관계 플레이트는 **서로 다른 형상 = 별도 금형 필요**
    - **SFMT Step B 업데이트**: 국소 합동 쌍 열거 시 변환군 크기 |T| = 12 → 6 로 축소. k_local 값이 감소 → **m_distinct 하한 강화** (더 많은 고유 형상 필요).
    - **예: 13S5P 13-셀 기준**: v0.2.15의 빨강 쌍(P1 ≅ P6, 60° 회전)은 엠보 규칙 하에서도 **동일 형상 유효** (회전만으로 합동, 거울 없음). 그러나 만약 "60° 회전 + 거울반사"로만 일치하는 쌍이 있었다면 엠보 규칙 하에서 **별개 형상**으로 분리.
    - **재고 관리 함의**: 좌우 대칭 배치처럼 보이는 팩에서도 실제로는 좌/우 플레이트가 거울상이면 **금형 2세트** 필요. 좌·우 동일 형상 사용하려면 팩 배치를 회전 대칭으로 설계해야 함.
    - **설계 가이드**: 합동 쌍 탐색 시 회전 대칭 우선(거울반사 배제) → 중앙 대칭(C2), 3회 회전 대칭(C3) 구성으로 플레이트 재사용 극대화.
22. **사각 정배열 + 컬럼 파티션은 m_distinct 하한이 결정적으로 도출된다** (사각 정배열 컬럼 파티션 최적성 정리) ★★★★ 최우선
    - **정리**: N = S × P 사각 정배열에서 각 그룹 G_i 를 i-번째 컬럼(P×1 P-오미노)으로 선택하면,
      - m_min = **1** (S 짝수) — 모든 플레이트가 2P×1 직사각 ㄷ자 쌍으로 합동, 단일 금형 ₩20M
      - m_min = **2** (S 홀수) — 양끝 I자 1종 + 내부 ㄷ자 쌍 1종, 2종 금형 ₩40M
    - **증명 개요**: (1) 컬럼 P×1 P-오미노 S개는 모두 합동 → m ≤ 1. (2) 직렬 병합 시 상·하면이 교대로 ㄷ자 쌍을 형성. S 짝수면 전 플레이트 2P×1 합동 → m = 1. S 홀수면 끝단 단독 I자(P×1)가 불가피 → m ≥ 2, 실제 달성 가능. ∎
    - **원칙 20·22 결합 경제효과**:
      - 13S5P 엇배열(8·8·10·13·13·13): m = 6, 금형 ₩120M, 실물 ₩150M
      - 13S5P 사각 정배열(13×5): m = 2, 금형 **₩40M**, 재료 +₩20K → 총 ₩60M
      - **절감 ₩80~90M (67%↓)**, 금형 재사용률 6.5× → 13×
    - **설계 권고 우선순위**:
      1. **S 짝수 사각 정배열** (단일 금형) — 케이스 공간만 허용되면 무조건 1순위
      2. **S 홀수 사각 정배열** (2종 금형)
      3. **S 짝수 엇배열** (공간 제약 시 — hex 펜토미노 ICC 해)
      4. **S 홀수 엇배열 / 계단형** (최후 수단, 6~8종 금형)
    - **실무 한계**: 케이스 폭·높이가 규격(UN38.3·UL·고객 사양)으로 고정된 경우 엇배열·커스텀 불가피. 신규 케이스 설계 단계에서는 반드시 원칙 22를 먼저 검토.
23. **배열 방식은 3-way 선택 — 정배열 / 엇배열 / 커스텀** (배열 선택 자유도 불변) ★★★
    - **모드 정의**:
      - `square` (정배열): S×P 정사각 격자, 피치 pitch_x = pitch_y = d + gap. 원칙 22 직접 적용.
      - `staggered` (엇배열): S×P 육각 격자, 짝수 행 offset_x = pitch/2, pitch_y = pitch·√3/2. 원칙 17 적용.
      - `custom` (커스텀): 행별 셀 수 rows = [n_1, n_2, …, n_R] 수동 지정 + 직렬 수 S 별도 설정.
    - **커스텀 입력 규약**:
      - rows: 쉼표 구분 정수 배열 (예: `8,8,10,13,13,13` = 6행, 총 N=65, 계단형 13S5P 실물 홀더)
      - S: 직렬 수 (사용자 수동). N/S 가 균등 정수면 자동 그룹 분할, 아니면 사용자 지정 매핑 필요
      - 피치: staggered와 동일 (hex 격자, 짝수 행 중앙정렬)
    - **니켈 패턴 매핑 전략**: 커스텀 모드에서 S-그룹 → 물리 셀 매핑은 다음 중 택1:
      1. **스네이크 자동**: 좌→우/위→아래 보스트로페돈으로 순차 분할 (원칙 12·13 인접성 자동 보장)
      2. **수동 지정**: 셀 클릭/좌표 입력으로 그룹 번호 직접 할당
      3. **DXF 업로드**: 홀더 CAD에서 셀 좌표·그룹 ID 직접 파싱
    - **실무 사용 사례**:
      - 유진로봇 AMR 팩: 케이스 내부 장애물 회피로 계단형 불가피 → 커스텀 모드 + 수동 그룹 지정
      - 제닉스로봇 UPS: 규격 케이스 사각 → 원칙 22 적용 S 짝수 정배열
      - 서진시스템 48V/100Ah LFP: S=16(16셀 직렬) 짝수 + 사각 정배열 → 단일 금형 ₩20M 권고

    - **역사적 맥락**: LINE 기반 니켈 렌더링 시절에는 "니켈이 셀 위에 있어야 셀에 가려지지 않는다"는 규칙이 필요했다.
    - **현재 적용 범주**: LINE 기반 니켈에만 해당. 채워진 폴리곤 방식에서는 원칙 27이 적용된다.
    - **폴리곤 방식에서의 변경 이유**: 채워진 폴리곤 니켈이 셀 위에 그려지면 셀 원형이 가려져 위치 파악이 불가능해짐. 따라서 폴리곤 방식에서는 셀을 상위 레이어로 이동.

25. **니켈 플레이트 형상 = [인접 셀 쌍 연결선 stroke=2R] + [셀 원형 배경 r=R]** ★★★ **절대 불변**
    - **물리 근거**: 니켈 스트립은 그룹 내 인접 셀들을 연결하는 금속 판. 연결된 셀들 사이의 공간을 정확히 채워야 한다.
    - **알고리즘**:
      ① grid-인접 쌍(|Δrow|+|Δcol|=1)마다 `<line stroke-width="2R" stroke-linecap="round">` 렌더
      ② 모든 셀 위치에 `<circle r="R" fill="plate_color">` 렌더
      ③ 셀(원칙 27 레이어 순서에 따라) 위에 일반 셀 심볼 렌더
    - **선 두께 2R의 의미**: 셀 직경과 동일 → 인접 셀 사이 공간을 빈틈없이 채우며, 동시에 인접 다른 그룹 셀과 겹치지 않음 (gap=0 기준으로 인접 셀 거리=2R)
    - **범용성**: 임의 배열(수천 가지)에 lookup table 없이 자동 적용. 셀 좌표와 그룹 정보만으로 항상 정확한 경계 보장.
    - **Convex Hull 방식 절대 금지**: 일렬 배치 그룹에서 hull=선분 → inflate(R) = capsule → 팩 전체 70%를 덮는 거대 도형 발생. 회복 불가.
    - **SVG 구현**: `<line stroke-width="{2R}" stroke-linecap="round"/>` per adjacent pair + `<circle r="{R}"/>` per cell

26. **니켈 플레이트 색상 = 주황/파랑 2색 교번 (plate 인덱스 기준)** ★★★ **절대 불변**
    - **규칙**: plate[0,2,4…] = 주황(#E07B26), plate[1,3,5…] = 파랑(#1A75C0)
    - **물리 근거**: 직렬 회로에서 인접 플레이트는 서로 다른 셀 극성에 연결된다. 2색 교번이 이 관계를 즉각적으로 시각화한다.
    - **위반 금지**: 단색(회색 등) 사용 절대 금지. 그룹 경계가 사라져 어느 셀이 어느 플레이트에 속하는지 구분 불가.
    - **outline**: 각 플레이트의 어두운 동일 계열 색(주황→#8C4200, 파랑→#08305A)으로 테두리 강조.

27. **레이어 순서 = 니켈(최하단) → 셀(중간) → 단자탭(최상단)** ★★★ **절대 불변**
    - **규칙**: SVG `<g>` 내 출력 순서: `buildNickel() → cells.join('') → buildTerminals()`
    - **근거**: 채워진 폴리곤 니켈은 셀 전체 영역을 덮는다. 셀이 아래에 있으면 가려져 위치 파악 불가. 셀은 반드시 플레이트 위에 렌더링되어야 한다.
    - **위반 금지**: 니켈을 셀 위에 그리면 모든 셀 위치 정보가 사라진다.
    - **원칙 24와 관계**: 원칙 24(LINE 기반 규칙)와 순서가 반대. 폴리곤 방식에서는 이 원칙(27)이 우선한다.

### 5×4 6S3P 정배열 열거 결과 (참고)

**단자 플레이트 형태 (G0 × G5):**

r0 |  531  448  420  455  679

- 내부 셀(r1c2=316, r2c2=327)이 가장 제약 많음.

### 원칙 12·13 적용 규칙

- **그룹 내 병렬 연결:** 같은 그룹의 셀 쌍은 반드시 상·하·좌·우 인접할 때만 니켈로 연결. 비인접 쌍은 니켈 없음(전기적으로 BMS/와이어 경유).
- **그룹 간 직렬 브리지:** G_i와 G_{i+1} 사이 인접한 셀 쌍이 존재할 때만 직렬 니켈 브리지 발생. 인접 쌍이 없으면 **배치 불가** 판정 (빨강 경고선 + "비인접 (점프 불가)" 라벨).
- **커스텀 격자 그룹 배정 알고리즘 (그리디):**
  1. 미배정 셀 중 row-major 순서 첫 번째를 seed로 선택.
  2. seed부터 시작해 P개 수집 시까지 확장:
     - 현재 그룹의 마지막 편입 셀의 인접 셀(우→하→좌→상 우선) 중 미배정인 것을 편입.
     - 후보가 없으면 이전 셀들의 인접 셀을 차순위로 탐색.
  3. 확장 실패 시 해당 격자는 **그룹 배정 불가** 판정.
- **효과:** 6S3P를 5×4 격자에 배치하면 G0=(r0,c0-c2), G1=(r0,c3-c4 + r1,c4) L자, G2=(r1,c1-c3) 등 L자·직선 혼재 배치. 일부 조합(예: 6S3P 5×4 직렬 G1→G2)은 인접 쌍이 없어 **라우팅 불가** 판정이 정답.

## 15. v0.3 확장 — 엇배열(Staggered / HCP) 지원 ★ (2026-04-14 신규)

### 15-1. 배경 및 실무 필요성

케이스 상하(세로) 공간이 협소한 경우 (예: 슬림 UPS, 저상형 AMR, 방송카메라 배터리), P방향 셀 배치를 엇배열(Hexagonal Close-Packed)로 바꿔 **팩 세로 높이를 약 13% 축소**한다. 동일 S·P 구성에서 체적 에너지밀도 상승.

### 15-2. 좌표 체계 확장 — cx[g][p] 2D화

```
pitch_y     = (arrangement === 'staggered') ? pitch × √3/2 : pitch
offset_odd  = (arrangement === 'staggered') ? pitch/2 : 0

cx[g][p] = margin + g × pitch + R + (p%2===1 ? offset_odd : 0)
cy[p]    = margin + p × pitch_y + R
```

- row 0 (p=0), row P-1 (p=짝수): 오프셋 0
- row 1, 3, 5... (p=홀수): +pitch/2 오프셋
- P=3 기준 수직 높이 축소: `R + 2×(√3/2)×pitch` ≈ 정방 대비 86.6%

### 15-3. 니켈플레이트 기하 변경

| 항목 | 정방(square) | 엇배열(staggered) |
|------|-------------|------------------|
| I자 세로 스파인 | 수직 rect | **지그재그 polyline** (stroke-width=nw, linejoin=round) |
| ㄷ자 좌·우 세로 | 수직 rect 2개 | 지그재그 polyline 2개 |
| ㄷ자 상·하 수평 브리지 | cy[0]/cy[P-1] 라인 | 동일 (row 0, row P-1은 오프셋 0이라 수평 유지) |
| 스파인 꺾임각 | 0° | tan⁻¹((pitch/2)/(pitch×√3/2)) = **30°** |

### 15-4. 단자 탭 위치 (staggered)

```
midP         = floor(P/2)
cx_ref       = cx[g][midP]   ← 중간 row 오프셋 반영
cy_center    = cy[midP]
```

P=3 → midP=1 → 중간 row (오프셋됨) 기준으로 탭 부착. 실제 케이스 탭 인출 위치와 일치.

### 15-5. 구현 파라미터 추가

```json
{
  "arrangement": "square"  // 또는 "staggered"
}
```

### 15-6. 실무 적용 체크리스트
